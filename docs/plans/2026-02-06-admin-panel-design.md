# Admin Panel Design

## Overview

A `/admin` route inside the existing DevRelCon NY app providing form-based CRUD for all Firestore collections. Built with Lit + Shoelace web components. Protected by domain-restricted Google auth (`@majorleaguehacking.com`).

Editors currently manage speakers, sessions, schedule, and all other conference data via raw Firebase console JSON editing. This admin panel replaces that with labeled forms, dropdowns, image uploads, and a proper schedule editor.

## Access Control

- `/admin` route checks the currently authenticated Firebase user
- If not signed in, shows the existing sign-in dialog
- If signed in, checks that `user.email` ends with `@majorleaguehacking.com`
- If unauthorized domain, shows "Access denied" message
- No allowlist to maintain — any MLH Google account gets in

## Navigation & Routes

Left sidebar nav with collection groups:

- **Content:** Speakers, Sessions, Schedule, Previous Speakers
- **Event:** Tickets, Partners, Team, Videos, Gallery
- **Site:** Blog, Config

Route structure:

- `/admin` — redirects to first section
- `/admin/{collection}` — list view
- `/admin/{collection}/new` — create form
- `/admin/{collection}/:id` — edit form
- `/admin/partners/:id/items` — subcollection list
- `/admin/partners/:id/items/new` — subcollection create
- `/admin/partners/:id/items/:itemId` — subcollection edit
- Same pattern for `team/:id/members`

## List Views

Each collection gets a Shoelace table showing key identifying fields:

| Collection        | Columns shown                   |
| ----------------- | ------------------------------- |
| Speakers          | name, company, order, featured  |
| Sessions          | title, tags, complexity         |
| Schedule          | date, dateReadable, track count |
| Tickets           | name, price, available, soldOut |
| Partners          | title, order, item count        |
| Team              | title, member count             |
| Previous Speakers | name, company, order            |
| Videos            | title, speakers, order          |
| Gallery           | url (thumbnail), order          |
| Blog              | title, published                |
| Config            | document ID                     |

Each list has:

- "New" button at top
- Edit and Delete buttons per row
- Delete shows a Shoelace confirmation dialog

For parent/child collections (partners → items, team → members), the list shows parent groups. Clicking into one shows its children with breadcrumb nav (e.g., `Admin > Partners > Gold Partners`).

## Form Architecture

A generic reusable form renderer takes a field schema and renders the appropriate Shoelace inputs.

Field type mapping:

| Data type                           | Shoelace component                    |
| ----------------------------------- | ------------------------------------- |
| `string`                            | `<sl-input>`                          |
| `number`                            | `<sl-input type="number">`            |
| `boolean`                           | `<sl-switch>`                         |
| `long text`                         | `<sl-textarea>`                       |
| `image`                             | Custom upload component (see below)   |
| `array of strings` (tags, speakers) | `<sl-select multiple>`                |
| `enum` (complexity, language)       | `<sl-select>` with predefined options |
| `array of objects` (socials)        | Repeatable field group component      |

Behavior:

- Save writes directly to Firestore — no draft/publish workflow
- Shoelace toast on success/error
- Generated collections (`generatedSpeakers`, `generatedSessions`, `generatedSchedule`) are auto-regenerated by existing Cloud Functions when raw collections are written to

## Schedule Editor

The most complex form. Editing a single day has three sections:

**1. Day info:**

- `date` — `<sl-input type="date">`
- `dateReadable` — `<sl-input>` (e.g., "July 17")

**2. Tracks:**

- Repeatable list of track title inputs
- Add/remove/reorder tracks
- Track order here defines column order on the frontend

**3. Timeslots:**

- Displayed as a list of cards
- Each timeslot has:
  - `startTime` / `endTime` — `<sl-input type="time">`
  - For each track: a searchable `<sl-select>` to pick a session from the `sessions` collection (shows session title)
  - An `extend` number field per session assignment (defaults to 1)
- "Add timeslot" button appends a new one
- Timeslots sorted by start time on save
- Sessions already assigned elsewhere show "(assigned)" label in the dropdown

## Image Uploads

A reusable Lit component for image fields:

1. Shows a drop zone / file picker button
2. On file select: uploads to Firebase Storage at `admin-uploads/{collection}/{timestamp}-{filename}`
3. Shows upload progress with `<sl-progress-bar>`
4. `optimize-images` Cloud Function triggers automatically on any Storage upload — resizes and optimizes the image, marks it with `optimized: true` metadata to prevent re-processing
5. Populates the URL field with the public Storage URL
6. Shows image preview thumbnail
7. If field already has a URL, shows existing image with a "Replace" button

Used for: speaker photos, speaker company logos, partner logos, team member photos, gallery images, blog featured images.

### Storage Rules Update

Current `storage.rules` blocks all writes. Update to allow authenticated MLH-domain users to write to `admin-uploads/`:

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /admin-uploads/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null
        && request.auth.token.email.matches('.*@majorleaguehacking[.]com$');
    }
    match /{allPaths=**} {
      allow read: if true;
      allow write: if false;
    }
  }
}
```
